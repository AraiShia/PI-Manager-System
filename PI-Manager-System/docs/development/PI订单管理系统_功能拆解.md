> 基于流程图梳理，版本：v1.0
> 日期：2026-04-29

---

## 一、系统概述

### 1.1 系统定位
企业内部订单管理系统（分部门管理），覆盖从**产品管理 → 客户/供应商管理 → 报价 → PI下单 → 采购 → 出货 → 收付款 → 库存**的全链路闭环。

### 1.2 核心价值
- **统一编号规则**：所有单据（商品、PI、采购单、报价单）通过编号串联
- **分部门管理**：多部门（维那/索英普/马迪那/银达）独立运营，数据隔离
- **全流程追溯**：从询价到回款，每个环节可查可控

### 1.3 编号规则汇总

| 单据类型 | 编号规则 | 示例 |
|---------|---------|------|
| 系统商品编号 | `部门编号 + 产品类别编号 + 年份 + 自然序号` | `SA01260001`（S部门+汽配件C01+26年+0001） |
| PI号 | `部门编号 + 客户编号 + 年月日 + 识别号` | `SA012612011`（S部门+A01客户+261201+1） |
| 采购单号 | `V + PI号 + 供应商号` | `VSA012612011-001` |
| 报价单号 | `Q + PI号` | `QSA012612011-1` |

**部门编号**：维那(W)、索英普(S)、马迪那(M)、银达(D)

**产品类别编号**：
- 汽配件(C)：发动机(01)、曲轴(02)、刹车片(03)、杂项
- 办公家具(F)：椅子类(01)、桌子类(02)、工程定制(88)
- 百货类(B)

### 模块4：报价单管理

#### 核心功能
向客户生成报价单，支持根据历史采购价快速报价。

#### 业务流程
```
选择客户 → 录入OE号/客户号/数量 → 系统取价（该客户最后一次采购价）→ 生成报价单
```

#### 字段清单

| 字段名 | 说明 | 必填 | 备注 |
|-------|------|-----|------|
| 报价单号 | 自动生成，格式：`Q + PI号逻辑` | ✅ | 如 QSA012612011-1 |
| 客户 | 关联客户档案 | ✅ | |
| 产品 | 关联产品档案 | ✅ | 支持多产品 |
| OE号/客户号 | 用于匹配 | ✅ | 可批量导入 |
| 数量 | 采购数量 | ✅ | |
| 单价 | 系统自动带出/可手动修改 | ✅ | |
| 总价 | 自动计算 | ✅ | |
| 有效期 | 报价有效期 | | |
| 付款条款 | | | |
| 备注 | | | |

#### 业务规则
- 默认取价：该客户对该产品的最后一次采购价
- 报价单号格式：`Q + PI号 - 序号`（-1, -2表示该PI下的第几个报价）
- 报价单可转为PI（确认后生成PI号）

#### 软件功能点
- [ ] 报价单列表（状态：草稿、已发送、已确认、已过期）
- [ ] 新建报价单（支持模糊搜索产品/OE号）
- [ ] 批量导入（Excel导入OE号+数量）
- [ ] 历史价格参考（展示该产品对该客户的历史报价）
- [ ] 转为PI（报价确认后一键生成PI）
- [ ] 导出PDF/发送邮件
- [ ] 报价单模板管理

### 模块5：PI管理（Proforma Invoice）

#### 核心功能
PI是整个系统的核心单据，作为唯一批次号串联所有流程。

#### 字段清单

| 字段名 | 说明 | 必填 | 备注 |
|-------|------|-----|------|
| PI号 | 唯一批次号 | ✅ | 格式：部门+客户+年月日+识别号 |
| 客户 | 关联客户 | ✅ | |
| 产品明细 | 关联产品+数量+单价 | ✅ | |
| 总金额 | 自动汇总 | ✅ | |
| 付款记录 | 定金、尾款1、尾款2 | | |
| 定金金额 | | | |
| 定金付款日期 | | | |
| 尾款1金额/日期 | | | |
| 尾款2金额/日期 | | | |
| 状态 | 待确认/已付定金/生产中/已出货/已完成 | | |

#### 业务规则
- PI号是唯一批次号，贯穿采购、出货、收付款
- 支持分阶段付款：定金 → 尾款1 → 尾款2
- PI确认后自动通知采购部门

#### 软件功能点
- [ ] PI列表（状态筛选：待确认/生产中/已出货/已完成）
- [ ] 新建PI（从报价单转入 或 直接新建）

- [ ] 付款记录录入（定金、尾款1、尾款2）
- [ ] 付款状态自动更新（定金到账后自动变更状态）
- [ ] PI复制（基于历史PI快速新建）
- [ ] 导出PI PDF
- [ ] 关联采购单、出货单、收付款记录

#### 规则定义

| 编号类型 | 规则 | 示例 |

## 五、开发优先级建议

### 第一期（核心闭环，MVP）
1. **产品管理** - 系统基石，最先开发
2. **客户管理** - 基础档案
3. **供应商管理** - 基础档案
4. **PI管理** - 核心单据
5. **采购管理（采购合同）** - 基于PI采购
6. **入库操作** - 采购入库

### 第二期（完整流程）
7. **报价单管理** - 报价→PI
8. **出货管理** - 出货→CI/PL
9. **收付款管理** - 收款+付款
10. **库存管理** - 实时库存

- 编号规则配置
- 审批流程
- 报表统计
