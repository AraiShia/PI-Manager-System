# 产品类目管理设计

## 1. 需求概述

在系统设置中提供产品类目管理功能，允许用户查看、新增、删除产品类目。

## 2. 用户选择

| 选项 | 选择 |
|------|------|
| 位置 | 系统设置中（SettingsDialog） |
| 存储方式 | 混合模式（本地+数据库） |
| 功能 | 查看、新增、删除（不提供编辑） |

## 3. 架构设计

### 3.1 数据存储策略

| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| **系统预设类目** | 数据库 | 发动机、曲轴等默认类目，代码固定 |
| **用户自定义类目** | 本地配置 | 用户新增的类目，可删除 |

### 3.2 加载流程

```
1. 从数据库加载预设类目列表
2. 从本地配置加载自定义类目列表
3. 合并两个列表并显示
4. 用户可见所有类目，但只有自定义类目可删除
```

### 3.3 保存流程

```
新增类目 → 保存到本地配置文件（不写入数据库）
删除类目 → 从本地配置移除（预设类目不删除）
```

## 5. 数据模型

### 5.1 类目数据结构

```python
class CategoryItem:
    code: str           # 类目代码
    name: str           # 类目名称
    is_system: bool     # 是否为系统预设
```

### 5.2 本地配置结构

```json
{
    "custom_categories": [
        {"code": "C01", "name": "自定义类目1"},
        {"code": "C02", "name": "自定义类目2"}
    ]
}
```

### 5.3 类目代码规则

- **系统预设代码**：数字开头（01, 02, 03...）
- **用户自定义代码**：`C` 开头 + 序号（C01, C02...）

## 6. 功能流程

### 6.1 新增类目

```
用户点击"新增类目" → 弹出输入对话框 → 用户输入类目名称
    → 点击确定 → 保存到本地配置 → 刷新列表显示新类目
```

### 6.2 删除类目

```
用户点击自定义类目的"删除"按钮 → 弹出确认对话框
    → 用户确认 → 从本地配置移除 → 刷新列表
```

### 6.3 保存设置

```
用户点击"保存" → 将自定义类目写入本地配置文件
    → 显示保存成功提示 → 关闭对话框
```

## 7. 后端 API

### 7.1 获取类目列表

```
GET /api/product-categories/

Response:
[
    {"code": "01", "name": "发动机", "is_system": true},
    {"code": "02", "name": "曲轴", "is_system": true},
    ...
]
```

### 7.2 新增类目（本地配置）

新增类目不通过 API，直接写入本地配置文件。

## 9. 验收标准

- [ ] 在系统设置中显示产品类目管理区域
- [ ] 类目列表显示所有类目（包括预设和自定义）
- [ ] 预设类目标记 `[系统]` 标识，不可删除
- [ ] 自定义类目显示 `[删除]` 按钮
- [ ] 新增类目保存到本地配置
- [ ] 删除类目仅从本地配置移除
- [ ] 点击保存后自定义类目被正确保存

## 10. 替代方案

### 方案 A：全部使用数据库存储

- 优点：数据统一管理
- 缺点：需要后端支持新增/删除 API

### 方案 B：全部使用本地配置

- 优点：实现简单
- 缺点：用户自定义类目不会同步到其他设备

**当前选择：方案 C（混合模式）**
- 预设类目在代码中定义，确保一致性
- 自定义类目在本地配置，保护用户数据
