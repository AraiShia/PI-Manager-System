# 产品管理总表 - 搜索功能合并设计

## 1. 需求概述

合并产品管理总表中两个重复的搜索输入框，统一搜索入口，简化界面。

## 2. 当前问题

产品管理总表存在两个搜索框，功能部分重叠：

| 组件 | 位置 | 触发方式 | 问题 |
|------|------|----------|------|
| `search_input` | 工具栏搜索区域 | 按钮搜索 | ❌ 已注释 |
| `product_search_input` | 筛选栏 | 实时搜索 | ❌ 与上面重复 |

用户容易困惑，不知道该用哪个搜索框。

## 3. 设计方案

### 3.1 目标布局

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│  产品列表                                                    [+ 新增产品]                │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│  [搜索框: OE号/产品编号/品牌...] [分类下拉] [搜索] [重置]  客户: [全部客户 ▼] [筛选]      │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────────────────────────────┐     │
│  │                               产品表格                                         │     │
│  └─────────────────────────────────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 改动清单

| 区域 | 改动 | 说明 |
|------|------|------|
| **工具栏搜索区域** | ✅ 恢复 `search_input` | 按钮搜索，触发 `search_products` |
| **工具栏搜索区域** | ✅ 保留 `category_filter` | 分类下拉筛选 |
| **筛选栏** | ✅ 保留 `product_customer_filter` | 客户筛选（下拉框） |
| **筛选栏** | ❌ 删除 `product_category_filter` | 与工具栏 `category_filter` 重复 |
| **筛选栏** | ❌ 删除 `product_search_input` | 与工具栏 `search_input` 重复 |
| **筛选栏** | ❌ 删除 `product_status_label` | 不再需要显示筛选状态 |

### 3.3 工作流程

**主搜索**（关键词 + 分类）：
1. 用户在搜索框输入关键词
2. 选择分类（可选）
3. 点击"搜索"按钮 → 调用后端API搜索（`search_products`）

**客户筛选**（补充条件）：
1. 用户在搜索结果基础上选择客户
2. 点击"筛选"按钮 → 前端本地过滤当前搜索结果
3. 或支持组合搜索：关键词 + 分类 + 客户同时生效

### 3.4 用户体验优化

- **统一入口**：搜索框只有一个，减少困惑
- **按钮控制**：用户主动触发搜索，避免频繁请求
- **灵活筛选**：客户筛选支持组合搜索，可与关键词+分类同时使用
- **清晰层次**：主搜索（关键词+分类）与补充筛选（客户）层次分明

### 4.3 搜索方法调整

修改 `search_products` 方法，支持组合搜索参数：

```python
def search_products(self):
    keyword = self.search_input.text().strip()
    category_id = self.category_filter.currentData()
    category_id = category_id if category_id != 0 else None
    customer_id = self.product_customer_filter.currentData()
    customer_id = customer_id if customer_id != 0 else None

    try:
        products = self.api_client.search_products(keyword, category_id, customer_id)
        self._populate_products_table(products)
    except Exception as e:
        print(f"搜索失败: {e}")
```

### 4.4 客户筛选方法

```python
def filter_by_customer(self):
    """根据客户筛选产品（在当前搜索结果中过滤）"""
    customer_id = self.product_customer_filter.currentData()
    if customer_id == 0:
        # 未选择客户，重新搜索
        self.search_products()
        return

    # 获取当前表格中的产品
    products = []
    for row in range(self.products_table.rowCount()):
        product_id = self.products_table.item(row, 0).data(Qt.ItemDataRole.UserRole)
        if product_id:
            products.append({'id': product_id, 'customer_id': customer_id})

    # 本地过滤
    filtered = [p for p in products if p.get('customer_id') == customer_id]
    self._populate_products_table(filtered)
```
