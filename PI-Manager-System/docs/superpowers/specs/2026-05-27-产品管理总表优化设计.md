## 一、需求概述

### 1.1 核心目标

优化现有产品管理模块，实现基于客户和类别的快速筛选功能，提升数据检索效率。

### 1.2 业务背景

- 产品数据按客户区分，每个客户有独立的 OE 号、编号、价格体系
- 当前产品列表缺少有效的筛选机制
- 需要支持按客户、类别、关键词多维度筛选

---

## 二、数据模型

### 2.1 核心表关系

```
客户产品 (prd_customer_product)
    │
    ├── customer_id → 客户 (crm_customer)
    ├── category_id → 产品类别 (prd_product_category)
    │
    ├── codes → 客户产品编号 (prd_customer_product_code)
    │       └── customer_code, is_primary
    │
    └── oes → 客户OE号 (prd_customer_product_oe)
            └── oe_number, is_primary
```

### 2.2 字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | INT | 主键 |
| customer_id | INT | 客户ID |
| category_id | STRING | 产品类别代码 |
| system_code | VARCHAR | 系统产品编号，格式：客户编号+部门+类别+年份+序号 |
| product_name | VARCHAR | 产品名称 |
| customer_model | VARCHAR | 客户型号 |
| color | VARCHAR | 颜色 |
| brand | VARCHAR | 品牌 |
| price_usd | DECIMAL | USD 价格 |
| price_rmb | DECIMAL | RMB 价格 |
| specifications | TEXT | 规格参数 |
| detail_desc | TEXT | 详细描述 |
| image_url | VARCHAR | 主图URL |
| is_active | BOOLEAN | 是否激活 |  ；；不需要显示

## 四、功能模块

### 4.1 筛选功能

**客户端筛选：**
- 首次加载所有数据，存入本地缓存
- 切换筛选条件时，在缓存数据中过滤
- 关键词搜索实时响应，300ms 防抖

**筛选优先级：**
1. 客户筛选（精确匹配）
2. 类别筛选（精确匹配）
3. 关键词搜索（模糊匹配：产品名称、OE号、编号）

**组合效果：**
- 客户 + 类别：显示该客户该类别的产品
- 客户 + 搜索词：显示该客户下匹配的产品
- 类别 + 搜索词：显示该类别下匹配的产品
- 三者组合：显示同时满足条件的产品

### 4.2 数据加载

**批量加载客户和类别：**
```python
# 首次加载时获取
customers = api_client.get_customers()  # 下拉框数据源
categories = api_client.get_product_categories()  # 下拉框数据源
products = api_client.get_customer_products()  # 完整产品列表
```

**数据缓存：**
- 存储在 `_product_cache` 变量
- 筛选操作在缓存数据上执行，不重复请求

### 4.3 编辑功能

**点击操作列编辑按钮：**
- 打开产品详情弹窗（复用现有弹窗）
- 支持编辑：基础信息、价格、规格、包装、客户产品编号、客户OE号

### 4.4 新建功能

**点击新建按钮：**
- 打开产品详情弹窗（空白表单）
- 必填字段：客户、产品名称
- 保存后刷新列表
