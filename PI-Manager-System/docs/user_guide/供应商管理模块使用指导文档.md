# 供应商管理模块使用指导文档

## 一、模块概述

供应商管理模块是PI订单管理系统的核心功能之一，用于管理供应商的基本信息、联系方式等。该模块支持供应商的新增、编辑、删除、批量导入等操作，并自动生成符合规范的供应商编号。

## 二、功能特性

### 2.1 核心功能
| 功能 | 说明 | 入口位置 |
|------|------|----------|
| 供应商列表 | 展示所有供应商信息 | 导航栏"供应商管理" |
| 新增供应商 | 创建新的供应商记录 | "+ 新增供应商"按钮 |
| 编辑供应商 | 修改已有供应商信息 | 列表"编辑"按钮 |
| 删除供应商 | 删除单个或多个供应商 | "删除选中"按钮 |
| 批量导入 | 从Excel/CSV文件批量导入 | "批量导入"按钮 |

### 2.2 供应商编码规则

供应商编码采用 **SP+省份代码+城市代码+三位数字** 的格式：

- **SP**：固定前缀，表示供应商(Supplier)
- **省份代码**：2位数字，采用GB/T 2260行政区划代码（如广东为44）
- **城市代码**：1位32进制字符（0-9, A-V），表示省内城市序号
- **序号**：3位数字，从001开始递增

**示例：**
- SP44L001：广东省云浮市第1个供应商
- SP111001：北京市第1个供应商
- SP13A005：河北省廊坊市第5个供应商

## 三、操作指南

### 3.1 查看供应商列表

1. 点击左侧导航栏的 **供应商管理** 选项
2. 系统将自动加载所有供应商数据
3. 列表展示字段：
   - 复选框（用于多选操作）
   - ID、供应商编号、供应商名称
   - 地区（省份+城市）
   - 联系人、电话
   - 操作（编辑按钮）

### 3.2 新增供应商

1. 点击供应商列表页面的 **+ 新增供应商** 按钮
2. 在弹出的对话框中填写以下信息：
   - **供应商名称**（必填）：输入供应商全称
   - **省份**（必填）：从下拉菜单选择省份
   - **城市**（必填）：选择省份后，城市下拉菜单自动更新
   - **联系人**（选填）：联系人姓名
   - **电话**（选填）：联系电话
   - **邮箱**（选填）：电子邮箱
   - **地址**（选填）：详细地址
3. 点击 **保存** 按钮完成创建
4. 系统自动生成供应商编号

### 3.3 编辑供应商

1. 在供应商列表中找到需要修改的供应商
2. 点击该供应商右侧的 **编辑** 按钮
3. 在弹出的对话框中修改相关信息
4. 点击 **保存** 按钮确认修改

**注意：供应商编号不可修改**

### 3.4 删除供应商

#### 单个删除
1. 在供应商列表中找到需要删除的供应商
2. 勾选该行最左侧的复选框
3. 点击上方的 **删除选中** 按钮
4. 在确认对话框中点击 **确定**

#### 批量删除
1. 勾选多个需要删除的供应商复选框
2. 或点击 **全选** 复选框选中所有供应商
3. 点击 **删除选中** 按钮
4. 在确认对话框中点击 **确定**

**注意：删除操作不可恢复，请谨慎操作**

### 3.5 批量导入供应商

1. 点击供应商列表页面的 **批量导入** 按钮
2. 在文件选择对话框中选择Excel文件（.xlsx/.xls）或CSV文件
3. 系统自动解析文件并导入数据

**文件格式要求：**

| 列名（支持智能识别） | 说明 | 是否必填 |
|----------------------|------|----------|
| 供应商/名称 | 供应商全称 | 必填 |
| 省份 | 省份名称（如：广东、北京） | 必填 |
| 城市 | 城市名称（如：广州、深圳） | 必填 |
| 联系人 | 联系人姓名 | 选填 |
| 电话/手机 | 联系电话 | 选填 |
| 邮箱/email | 电子邮箱 | 选填 |
| 地址 | 详细地址 | 选填 |

**导入结果说明：**
- 成功数量：成功导入的供应商数量
- 失败数量：导入失败的记录数量
- 失败原因：可能包括省份/城市不匹配、缺少必填字段等

## 四、数据结构

### 4.1 供应商主表（sup_supplier）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键ID |
| dept_id | String(10) | 部门标识（S/W/M/D） |
| supplier_code | String(50) | 供应商编号（唯一） |
| supplier_name | String(200) | 供应商名称 |
| region | String(100) | 地区（省份+城市） |
| status | Integer | 状态（1=启用，0=禁用） |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 4.2 供应商联系表（sup_supplier_contact）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键ID |
| supplier_id | Integer | 关联供应商ID |
| name | String(100) | 联系人姓名 |
| phone | String(50) | 联系电话 |
| email | String(100) | 电子邮箱 |
| address | Text | 详细地址 |
| is_primary | Integer | 是否主联系人 |

## 五、API接口

### 5.1 接口列表

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取供应商列表 | GET | /api/suppliers | 获取所有供应商 |
| 获取单个供应商 | GET | /api/suppliers/{id} | 根据ID获取供应商 |
| 新增供应商 | POST | /api/suppliers | 创建新供应商 |
| 更新供应商 | PUT | /api/suppliers/{id} | 更新供应商信息 |
| 删除供应商 | DELETE | /api/suppliers/{id} | 删除供应商 |
| 批量导入 | POST | /api/suppliers/batch | 批量创建供应商 |
| 获取省份列表 | GET | /api/suppliers/provinces | 获取所有省份 |
| 获取城市列表 | GET | /api/suppliers/cities/{province} | 根据省份获取城市 |

### 5.2 请求示例

**新增供应商：**
```json
POST /api/suppliers
{
    "supplier_name": "广州市XX贸易有限公司",
    "province": "广东",
    "city": "广州",

        }
```

## 六、注意事项

1. **省市匹配**：选择省份后必须选择对应的城市，否则无法保存
2. **编码唯一性**：供应商编号由系统自动生成，确保唯一性
3. **数据完整性**：供应商名称、省份、城市为必填项
4. **批量导入**：建议先在Excel中整理好数据再导入，确保省市名称正确
5. **删除操作**：删除供应商会同时删除其关联的联系人信息，请谨慎操作
6. **部门隔离**：不同部门登录时只能看到本部门的供应商数据

## 七、故障排除

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无法新增供应商 | 缺少必填字段 | 检查供应商名称、省份、城市是否填写 |
| 城市下拉为空 | 未选择省份 | 先选择省份，城市列表会自动更新 |
| 批量导入失败 | 文件格式不正确 | 检查列名是否正确，确保必填字段有数据 |
| 删除失败 | 存在关联数据 | 确保该供应商没有关联的产品或订单 |
| 供应商编号重复 | 手动修改过编号 | 不要手动修改供应商编号 |

### 7.2 错误码说明

| 错误码 | 说明 |
|--------|------|
| 400 | 请求参数错误 |
| 404 | 供应商不存在 |
| 500 | 服务器内部错误 |
